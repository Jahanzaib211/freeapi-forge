Architecture

How it works

Forge Studio sits between your applications and LLM providers, adding routing, monitoring, and budget controls.

Your App │ ▼ Forge Studio (:5051) │ ├── /v1/chat/completions (OpenAI-compatible API) ├── /api/stream/chat (SSE streaming) ├── /mcp/sse (MCP tool server) ├── /api/trpc/* (typed UI operations) │ ├── Custom Provider Lookup │ └── Match model → direct proxy to provider API │ └── LiteLLM Fallback └── LiteLLM (:5050) → Groq / Gemini / Mistral / Cerebras / ...
API

Drop-in compatible

Works with any tool that speaks the OpenAI API format. Just change the base URL.

curl 
# Chat completions
curl http://localhost:5051/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "forge-chat",
    "messages": [{"role": "user", "content": "Hello"}]
  }'

# List models
curl http://localhost:5051/v1/models

# Health check
curl http://localhost:5051/health
python 
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:5051/v1",
    api_key="any-key"
)

response = client.chat.completions.create(
    model="forge-chat",
    messages=[
        {"role": "user",
         "content": "Hello from Forge"}
    ]
)

print(response.choices[0].message.content)
